Customer Support Operations Benchmark Environment
1. Problem Statement
"Build a complete OpenEnv benchmark environment simulating customer support ticket triage and resolution, with 3 progressively harder tasks, deterministic graders scoring 0.0-1.0, meaningful reward shaping across trajectories, reproducible baseline inference, and full Docker/HF Spaces deployment compliance."

2. Core Value Proposition
First standardized OpenEnv benchmark for production customer support operations - real human workflow combining classification, policy lookup, evidence handling, escalation decisions, and resolution under business constraints.

Why this matters: Companies spend billions on support operations. Agent performance here = real ROI.

3. Agent Interaction Flow
text
reset(task_id="refund_triage") 
→ observe queue 
→ classify 
→ lookup policy 
→ check order status 
→ draft resolution 
→ resolve/escalate 
→ graded score [0,1]
4. OpenEnv Interface
text
reset(task_id: str, seed: int) → Observation
step(action: Action) → StepResult(observation, reward, done, info)  
state() → State

7. Observation Space
python
class Observation(BaseModel):
    task_id: str                    # "refund_triage", "damaged_goods", "queue_priority"
    task_description: str           # Human-readable objective
    tickets: List[Ticket]           # Current queue
    retrieved_records: dict         # Policy/order/account data
    action_history: List[ActionSummary]
    step_count: int
    max_steps: int
    summary_text: str               # LLM-friendly paragraph
8. Tasks (Easy → Medium → Hard)
Task	Difficulty	Objective	Grader Criteria	Expected Baseline
Refund Triage	Easy	Single ticket refund eligibility	Policy lookup + correct resolution	0.7-0.8
Damaged Goods	Medium	Evidence handling + replacement policy	Missing info detection + escalation	0.4-0.6
Queue Priority	Hard	5-ticket queue under step budget	Prioritization + multi-ticket completion	0.2-0.4
Grader Requirements:

Deterministic: Same actions → same score

Normalized: Always [0.0, 1.0]

Breakdown: {"score": 0.85, "components": {...}}

9. Reward Shaping
Episode reward = trajectory progress + final grade:

text
Per-step rewards:
+0.15 correct classification
+0.15 relevant lookup  
+0.10 appropriate info request
-0.10 redundant lookup
-0.25 policy violation  
+1.00 final grader score [0,1]

Max episode: ~2.5 (diminishing returns prevent gaming)
